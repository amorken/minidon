# Plan: Initial Scaffolding for `minidon`

A Mastodon public-timeline streaming web app with a Go backend, in-memory recent buffer,
MeiliSearch-backed bulk index, and a TypeScript single-page frontend served from the Go binary.

This plan covers ONLY the initial scaffolding, README, and a draft architecture
document. No business logic will be implemented yet.

### Layout rationale

- `cmd/minidon` keeps the binary entrypoint thin and idiomatic.
- `internal/` prevents accidental external imports and groups concerns by capability
  (mastodon, ingest, buffer, index, api, static, model).
- `web/` is a self-contained Node project so frontend tooling does not pollute the Go
  workspace; its `dist/` is consumed via `go:embed` from `internal/static`.
- `deploy/` keeps containerization separate from source.
- `docs/` is the obvious home for the architecture document and future API docs.

## 4. Draft Architecture Document (`docs/architecture.md`)

Sections planned:

1. **Goals & non-goals** вҖ” Stream public timeline; expose recent + searchable history;
   single-binary deployment. Non-goals: posting, auth, multi-user, federation.
2. **High-level diagram** (ASCII):
   ```
   Mastodon instance в”Җв”Җstreaming APIв”Җв”Җв–¶ ingest в”Җв”Җв–¶ ring buffer (RAM)
                                            в”Ӯ
                                            в””в”Җв”Җв”Җв”Җв–¶ MeiliSearch (bulk index)
                                                       в–І
   Browser SPA в”Җв”ҖHTTP/SSEв”Җв”Җв–¶ Go API в”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”Җв”ҳ
                                в”Ӯ
                                в””в”Җ embedded static SPA
   ```
3. **Components**
   - **Mastodon client** (`internal/mastodon`): connects to
     `wss://<instance>/api/v1/streaming` (public) plus a periodic fallback to
     `GET /api/v1/timelines/public` for backfill on reconnect.
     - Recommend `mattn/go-mastodon` for the WebSocket streaming wrapper, behind an
       interface so a hand-rolled REST/WS client can be swapped in.
   - **Ingest pipeline** (`internal/ingest`): fan-out from a single source channel to
     (a) ring buffer, (b) MeiliSearch batched writer (debounced, e.g., flush every
     1 s or 100 docs), (c) live SSE subscribers.
   - **Ring buffer** (`internal/buffer`): bounded slice keyed by status ID, default
     500 items, O(1) recent-N reads, lock with `sync.RWMutex` (or `atomic.Pointer`
     swap of immutable snapshot for read-heavy access вҖ” decide later).
   - **Index** (`internal/index`): MeiliSearch primary index `statuses`; searchable
     attributes: `content`, `account.acct`, `account.display_name`, `tags.name`;
     sortable: `created_at`; filterable: `language`, `tags.name`.
   - **HTTP API** (`internal/api`):
     - `GET /api/timeline?limit=N` вҶ’ most-recent N from buffer
     - `GET /api/search?q=&limit=&offset=` вҶ’ MeiliSearch passthrough
     - `GET /api/stream` вҶ’ SSE of new statuses
     - `GET /healthz`, `GET /readyz`
     - `GET /*` вҶ’ embedded SPA (SPA fallback for client-side routing)
   - **Static assets** (`internal/static`): `go:embed web/dist` served via
     `http.FS`, with `index.html` fallback.
4. **Data model** вҖ” Subset of Mastodon Status that we persist/serve: `id`, `uri`,
   `url`, `created_at`, `content` (sanitized HTML or text), `language`,
   `account` (acct, display_name, avatar), `media_attachments` (preview_url, type),
   `tags`, `reblog` (recursively, depth 1).
5. **Frontend** вҖ” Vite + React + TypeScript. Recommend React for ecosystem familiarity;
   note Preact as a drop-in lighter alternative. State via React hooks + a tiny
   store (Zustand or context). EventSource for `/api/stream`. Initial views: live
   timeline, search box, status detail modal.
6. **Build & deployment** вҖ” Multi-stage Dockerfile (node:20-alpine вҶ’ golang:1.22 вҶ’
   gcr.io/distroless/static). Compose brings up `minidon` + `meilisearch` with a
   shared network and named volume. Cross-platform: pure Go, no cgo; frontend is
   plain Node.
7. **Configuration** вҖ” 12-factor env vars (table).
8. **Observability** вҖ” `slog` structured logs; `/metrics` Prometheus endpoint
   reserved for later; health endpoints from day one.
9. **Open questions / tradeoffs**:
   - Streaming via go-mastodon vs raw WS вҖ” depends on stability of WS wrapper.
   - Buffer dedup strategy on reconnect.
   - Whether SSE or WebSocket is preferable for the browser stream (lean SSE for
     simplicity).
   - MeiliSearch vs alternatives (Bleve embedded, Typesense, SQLite FTS5) вҖ” short
     comparison table.
10. **Future work** вҖ” auth, multiple instances, persistent buffer, rate-limit handling,
    media proxying/caching.
